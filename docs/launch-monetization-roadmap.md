# WAZA KIMURA — 一般公開 & マネタイズ ロードマップ

前提（オーナーの方針）:
- **1人で運営**する副業プロジェクト。継続的に手間がかかる施策はやらない。
- **自動化できるものは積極的にやる**。一度仕込んだら放置できる形が理想。
- 現状すでに本番運用中（自分＋α）。i18n（日英）実装済み。

---

## 現状の技術的事実（2026-06 時点のコード調査より）

| 項目 | 状態 | 公開時のリスク |
|---|---|---|
| `/api/ai-summary`（Gemini動画解析） | **オーナー限定**（Firebase IDトークン検証 `verifyOwner`） | なし（保護済み） |
| `/api/ai-tag` `/api/ai-group`（Claude API） | **認証なし・レート制限なし** | 🔴 誰でも叩けてAnthropic課金が青天井 |
| `/api/yt-search` 等（YouTube API） | **認証なし** | 🟡 無料クォータ(10,000units/日)を第三者に食い潰される |
| `/api/rss-proxy` `/api/thumb-proxy` | 認証なし（ただし外部API課金なし） | 🟢 CF Workers無料枠の消費のみ |
| Firestore | ユーザーデータ保存。セキュリティルールはリポジトリ外 | 🟡 公開前に「自分のドキュメントのみ読み書き可」を要確認 |
| インフラ費 | CF Workers 無料枠 100k req/日、Firestore Spark 無料枠 | 数百ユーザー規模までほぼ¥0 |

**結論: 「公開」自体より先に、APIコストの蛇口を締めるのが唯一のブロッカー。**

## フェーズ0 — 公開前の守り（必須・実装1〜2日相当）

1. **AI系エンドポイントにログイン必須化＋利用上限**
   - `/api/ai-tag` `/api/ai-group` に `verifyOwner` と同方式のIDトークン検証を入れる（オーナー限定ではなく「ログイン済みなら誰でも」に緩和した `verifyUser` を作る）。
   - Workers KV でユーザー毎の日次カウンタ（例: AIタグ提案 20回/日、AI要約は当面オーナーのみ継続）。
   - 未ログインユーザーはAI機能ボタン自体を非表示（手動タグ付けは全機能使える）。
2. **YouTube検索のクォータ防衛**: 同じくログイン必須化＋検索結果をKVで短期キャッシュ（同一クエリの再課金を防ぐ）。
3. **Firestoreセキュリティルール確認**: `request.auth.uid == userId` のみ読み書き可になっているか。なっていなければ修正（これはコードでなくFirebaseコンソール作業）。
4. **利用規約・プライバシーポリシー静的ページ**（1回書けば終わり）:
   - Googleログインで取得する情報（メール）、Firestoreに保存するデータ、AI機能で動画メタデータを外部API（Anthropic/Google）に送ること、を明記。
   - 「データの永続性を保証しない」免責も入れる（個人運営の防御として重要）。
5. **Firebase Blaze移行＋予算アラート**: Sparkのままだと成長時に突然書き込み失敗する。Blazeにして「月$10超えたらメール」を設定する方が、無料枠死守より手間がない。

## フェーズ1 — ソフトローンチ（無料公開・計測期間 1〜3ヶ月）

- 全機能無料（AI要約のみオーナー限定のまま）。まず「使われるか」を確かめる。
- 計測は既存インフラで足りる: Cloudflareのアクセス解析＋Firebase Authのユーザー数。**新たな分析ツールは入れない**（手間ゼロ方針）。
- アプリ内に1つだけ導線を置く: 「フィードバック / Buy me a coffee」リンク（設定画面の隅でよい）。

### 判断基準（フェーズ2に進むかどうか）
- 30日後にアクティブユーザー（自分以外）**10人未満** → マネタイズは保留し、無料のまま趣味運用継続（それも成功の一形態）。
- **10〜50人** → 寄付リンク＋BYOキー（後述）で様子見。
- **50人以上 or AI機能の開放要望が複数** → フェーズ2（有料プラン）へ。

## フェーズ2 — マネタイズ（従量コストに連動した freemium）

### 推奨モデル: 「AI機能だけ有料」のfreemium
理由: このアプリで**限界費用が発生するのはAI機能だけ**。手動のタグ・ノート・ビューはユーザーが増えてもほぼ¥0なので無料で配り、コストが乗る機能に課金を対応させると、赤字構造が原理的に生まれない。

| プラン | 内容 | 価格案 |
|---|---|---|
| Free | 動画管理・タグ・ノート・カスタムビュー全部。AIタグ提案 5回/月お試し | ¥0 |
| Pro | AIタグ提案 無制限（内部上限あり）＋AI動画要約・一言解説・分岐抽出 | ¥500/月 or $4.99/月 |

### 決済は Stripe Payment Links（運用手間ほぼゼロ）
- 管理画面でリンクを1本作るだけ。カート・請求書・解約はStripeが全部やる。
- Webhook（`checkout.session.completed` / `customer.subscription.deleted`）を `_worker.js` に1エンドポイント追加し、Firestoreのユーザードキュメントに `pro: true/false` を書くだけで自動化完結。
- 確定申告時はStripeのダッシュボードから年間売上をエクスポートするだけ。

### 併用オプション
- **BYOキー（自分のGemini APIキーを設定画面に貼る）**: Gemini無料枠は個人利用なら十分。ヘビーユーザーが自腹でAIを使え、こちらのコストはゼロ。無料プランの逃げ道として実装しておくと「金取るのか」批判の緩衝材にもなる。
- **寄付リンク（Buy Me a Coffee）**: フェーズ1から置いてよい。期待値は低いが手間もゼロ。

### やらないこと（理由つき）
- **広告**: 練習中に開くツールにバナーはUX毀損が大きく、数百人規模では月数百円にもならない。割に合わない。
- **買い切り**: AIの従量コストは毎月出るのに収入が一度きりになり、長期でコスト逆転する。
- **B2B（道場向けプラン）等**: 営業・サポートという「継続的な手間」の塊なのでやらない。

## 宣伝プラン — 「一度打って放置」型のみ

継続的なSNS運用・コミュニティ管理は方針に反するのでやらない。**打ちっぱなしで検索やコミュニティに残り続ける施策**に絞る。

### 一度きりの投稿（合計数時間の作業）
1. **Reddit r/bjj**（会員数百万・英語圏最大のBJJコミュニティ）: 「I built a free tool to organize BJJ instructionals」的なShow-off投稿。i18n済みなのでそのまま出せる。宣伝規約があるので通常投稿1回＋あとはコメントで聞かれたら答える程度に。
2. **X（日本語）**: 柔術アカウントのハッシュタグ（#ブラジリアン柔術 #柔術）で1投稿。デモ動画（画面録画30秒）を付ける。
3. **note / Zenn に開発記事**: 「柔術の教則動画管理アプリを個人開発した話」。個人開発系の記事は検索で長く生き続け、日本語圏の流入源になる。
4. **Product Hunt**（任意）: 英語圏の個人開発者向け。1回出すだけ。

### 仕込んで放置するSEO資産（1回の実装で恒久効果）
- ランディングページ: 現状アプリ本体しかないなら、`/about` 的な静的1ページ（スクショ・機能一覧・日英）を用意。OGPタグ・meta description も整備。
- タイトルを検索語に寄せる: 「BJJ video organizer」「柔術 教則 管理」等が拾われる文言をLPに置く。

### 自動化できる運用
- **エラー監視**: Cloudflare Workersのログ→重大エラー時のみメール通知（Logpushや簡易Webhook）。「静かなら見ない」運用。
- **フィードバック収集**: アプリ内フォーム→Firestoreに書くだけの1機能。通知はGmailフィルタで週次確認。
- **PWA更新告知**: 既存のバージョン表示（`<title>`）で足りる。リリースノートは書かない（手間削減）。

### やらないこと
- YouTuber/インフルエンサーへの営業（高手間・低確度）
- 継続的なSNS投稿・コミュニティ運営
- 有料広告（ニッチすぎてCPAが合わない）
